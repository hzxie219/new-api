## 注意事项

### DFX设计要点
1. **串行执行**：严格按顺序执行7个DFX维度,后续设计依赖前序结果
2. **红线优先**：安全性和可靠性是红线项,必须完整设计
3. **格式规范**：FMEA分析严格遵循references/specifications/FMEA分析输出格式定义.md
4. **实用性优先**：DFX设计要实用可落地,避免空洞理论

### 可靠性设计要点(FMEA)
1. **功能要求可量化**：如"3秒内返回结果",不能写"快速返回"
2. **失效模式四类**：失效、超时、慢、不完整
3. **失效原因三类**：输入、执行、输出
4. **改进措施实用**：重点保证业务成功(重试、监控、降级)
5. **不过度发散**：失效影响不要过度发散(如不要扩展到"客户流失")

### 安全设计要点(STRIDE)
1. **六类威胁全覆盖**：欺骗、篡改、否认、信息泄露、拒绝服务、权限提升
2. **防护措施具体**：不能写"加强安全",要写"AES-256加密+访问控制"
3. **影响等级明确**：高/中/低,并说明判断依据
4. **验证方法可执行**：如"安全扫描"、"渗透测试"

### 可测试性设计要点
1. **依赖解耦**：使用抽象接口和Mock,降低测试依赖
2. **覆盖率目标**：明确覆盖率目标(>80%)
3. **测试数据**：提供测试数据准备方法
4. **自动化支持**：设计自动化测试方案

### 可运维性设计要点
1. **黄金指标**：Latency、Traffic、Errors、Saturation必须覆盖
2. **告警分级**：P0-P3分级明确,处理时效明确
3. **运维工具**：提供实用的运维脚本和工具
4. **故障预案**：提供常见故障的应急处理方法
