# 概要设计总结与变更管理生成命令 (Chapter 7/9/10)

## 命令描述
生成模块概要设计说明书的第7章"总结"、第9章"变更控制"和第10章"修订记录"。这些章节主要负责设计文档的总结归纳、版本管理和变更追踪，确保设计文档的完整性和可追溯性。

**注意**: 本命令不生成第8章"业务逻辑相关的测试用例"，该章节由专门的测试设计命令负责。

## 命令语法
```bash
/chapter7-10 $1
```

## 参数说明

- **$1**: 必填参数，需求分析文档路径，指向需求分析文档

## 输出内容

**完成设计文档章节7/9/10**:

```
7. 总结
├── 7.1. 关联分析
└── 7.2. 遗留问题解决

9. 变更控制
└── 9.1. 变更列表

10. 修订记录
```

## 输出文件

追加到现有设计文档:`doc/design/tech_design.md`

## 注意事项

### 关联分析要点
- **全面识别**: 必须全面识别所有受影响的模块、版本、产品
- **准确评估**: 影响评估要准确，不能遗漏风险点
- **沟通确认**: 跨模块、跨产品影响必须与相关负责人沟通确认
- **提供方案**: 对每个影响点都要提供应对措施

### 遗留问题管理
- **问题必有方案**: 不允许存在无解决方案的问题
- **延期需说明**: 延期问题必须说明原因和计划
- **跟进要到位**: 延期问题要明确责任人和跟进计划
- **状态要明确**: 问题状态要清晰（已解决/部分解决/延期/转需求）

### 变更控制原则
- **变更必评审**: 所有变更必须经过评审确认
- **影响要分析**: 变更影响分析要全面
- **记录要完整**: 变更记录要包含完整信息
- **可追溯性**: 变更要能追溯到具体章节

### 版本管理规范
- **版本号规范**: 遵循统一的版本号规则
- **修订必记录**: 重要修订都要记录
- **信息要完整**: 包含作者、日期、修订内容
- **里程碑标注**: 重要版本要标注里程碑
