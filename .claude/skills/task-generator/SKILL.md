# Task Generator - 任务生成器

## 概述

Task Generator 帮助将详细设计文档转换为结构化的可执行任务清单（feature-task.md），用于指导 AI 进行编码实现。

## 输入输出

- **输入**：详细设计文档内容（由调用方提供，如通过命令参数或上下文传入）
- **输出**：可执行任务清单（写入调用方指定的位置）


## 工作流程

```
读取设计文档 → 分析功能点 → 任务拆解 → 依赖分析 → 生成 feature-task.md
```

## 阶段二：任务拆解

### 拆解规则

#### 规则 1: 数据结构跟随功能

- 数据结构定义**不单独**作为 Task
- 数据结构应放在**使用它的功能 Task** 中
- 原则：谁定义、谁使用、谁负责

#### 规则 2: 一个 Task 一个目标

- 每个 Task 只完成**一个独立的功能目标**
- 不相关的功能**必须拆分**成独立的 Task
- 判断标准：能否用一句话描述这个 Task 的目标？

#### 规则 3: 按功能内聚拆分

- **错误**：按代码层次拆分（数据层 → 逻辑层 → 接口层）
- **正确**：按功能内聚拆分（功能A → 功能B → 各自的集成）
- 依赖顺序：基础设施 → 业务逻辑 → 业务编排 → 接口层

### 任务类型分类

| 类型 | 描述 | 示例 |
|------|------|------|
| **基础设施** | 可复用的通用组件 | CSV 日志模块、HTTP 客户端 |
| **业务逻辑** | 核心业务处理（含相关数据结构） | 缓存管理、推理逻辑 |
| **业务编排** | 流程编排和协调 | Service Handler |
| **接口层** | API Handler | 实现具体的 HTTP Handler |
| **集成配置** | 路由注册、配置集成 | 注册路由、添加中间件 |
| **运维支持** | 监控、调试支持 | pprof、健康检查 |

## 阶段三：生成 feature-task.md

使用模板：[task-template.md](templates/task-template.md)

### 重要规则

> **必须遵循**：
> - **必须**标注任务之间的依赖关系
> - **不需要**写详细的实现步骤和验收命令
> - 任务描述应简洁明确，便于 AI 理解执行
