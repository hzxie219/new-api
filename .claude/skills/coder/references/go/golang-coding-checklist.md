## 01. 风格规范

### 1.1 遵从惯例
- [ ] 遵从Go语言社区约定俗成的行为习惯

### 1.2 文件行数
- [ ] 单个文件长度尽量不超过800行

### 1.3 缩进、括号、空格
- [ ] 【强制】使用gofmt工具处理代码格式
- [ ] 【强制】使用tab缩进
- [ ] 【强制】左大括号不换行
- [ ] 【强制】运算符和操作数之间留空格（作为输入参数或数组下标时除外）

### 1.4 标识符命名
- [ ] 【强制】采用MixedCaps/mixedCaps规则，首字母根据访问控制大小写
- [ ] 【强制】导出接口使用大写开头，非导出接口使用小写开头
- [ ] 【强制】缩写或特殊术语全大写或全小写（如IPServer/ipServer）
- [ ] 【强制】标识符不携带包名上下文信息（如http.Server而非http.HTTPServer）
- [ ] 【强制】包名不能和标准库重名
- [ ] 包名简短且只包含小写字母，不包含下划线（自动生成代码或_test后缀除外）
- [ ] 保持包名和目录一致
- [ ] 包名不使用常用的局部变量名
- [ ] 【强制】文件名使用小写字母，以下划线分隔单词
- [ ] 【强制】变量/常量名不以类型作为前后缀
- [ ] 未导出的全局变量/常量使用"_"前缀（未导出的错误值err除外）
- [ ] 【强制】错误变量名以Err/err开头
- [ ] 【强制】错误类型名以Error结尾
- [ ] 【强制】接受者命名短小（1-2个字母）、类型缩写、始终一致

### 1.5 编码格式
- [ ] 【强制】代码文件使用UTF-8编码格式
- [ ] 【强制】换行符使用unix格式

## 02. 注释

### 2.1 通用规则
- [ ] 单行注释符号与内容之间用一个空格隔开
- [ ] 每个可导出的名字都有注释说明
- [ ] 注释采用双斜杠（//），多行注释（/* */）仅用于包级别文档
- [ ] 注释符合godoc规范

### 2.2 声明注释
- [ ] 每个包都有包注释和版权说明（复杂包可使用doc.go）
- [ ] 函数/方法有注释说明功能、参数、返回值
- [ ] 结构体/接口有注释说明功能
- [ ] 全局变量有注释说明
- [ ] 大块常量或变量定义有总体和详细注释
- [ ] 类型别名有注释说明
- [ ] 行尾注释保持对齐一致

## 03. 错误处理

### 3.1 返回错误
- [ ] 【强制】错误信息作为最后一个返回值，不使用内联错误方式
- [ ] 使用context.Context的函数应返回error
- [ ] 【强制】遵守Fail Fast原则，出现失败立即返回error

### 3.2 日志
- [ ] 使用Key-Value日志库时，将上下文变量放到context中

### 3.3 错误处理
- [ ] 【强制】显式处理error或使用空白标识符忽略
- [ ] 忽略错误时添加注释说明
- [ ] 【强制】立即处理error，采用独立错误流，减少嵌套
- [ ] 每个错误只处理一次

### 3.4 错误字符串
- [ ] 错误字符串首字母小写且不加结束标点符号
- [ ] 导出名称、专有名词、首字母缩写词开头时可大写

### 3.5 类型断言
- [ ] 【强制】类型断言使用comma ok样式

### 3.6 panic/recover处理
- [ ] 【强制】panic必须在当前Goroutine被捕获
- [ ] 【强制】recover必须在defer中使用
- [ ] 避免滥用panic/recover，优先返回错误

## 04. 资源管理

### 4.1 主动关闭资源
- [ ] 【强制】资源使用完毕后确保已关闭，使用defer释放资源
- [ ] 【强制】资源管理存在失败时，先判断错误再defer释放
- [ ] 【强制】禁止在循环中直接使用defer

### 4.2 内存泄漏
- [ ] 【强制】避免循环引用与runtime.SetFinalizer结合使用
- [ ] 从大切片获取小切片时考虑使用copy代替re-slice
- [ ] 瞬间大流量场景考虑map内存占用问题

### 4.3 并发
- [ ] 【强制】禁止在闭包中直接调用循环变量
- [ ] 【强制】map和slice多goroutine读写时有适当同步机制
- [ ] 单例模式使用sync.Once避免重复执行

## 05. 项目组织

### 5.1 单元测试
- [ ] 【强制】项目提供单元测试（使用Testing/Gomock/GoStub/GoConvey）
- [ ] 【强制】执行go test时开启-race选项
- [ ] 性能分析或调试时指定-gcflags="-l"参数

### 5.2 模块化
- [ ] 项目提供mod文件
- [ ] 模块ID格式：sangfor.com/<部门或产品英文缩写>/<模块名称>

### 5.3 包导入
- [ ] 【强制】使用goimports规范包导入
- [ ] 【强制】禁止使用相对路径导入
- [ ] 【强制】非测试文件禁止使用.简化导入
- [ ] 【强制】强制使用分行导入
- [ ] 导入包分三组：标准库包、外部包、内部包，用空行区分
- [ ] 避免滥用空白导入（_），使用时添加注释
- [ ] 避免导入别名，除非有冲突

## 06. 函数

### 6.1 函数参数
- [ ] 同种类型参数放在相邻位置
- [ ] 尽量用值传递，非指针传递
- [ ] 需要修改参数、包含sync.Mutex、struct/array很庞大时使用指针
- [ ] 固定参数考虑实现多个函数
- [ ] map/slice/chan/interface类型不传递指针
- [ ] 不使用数组作为参数，使用切片

### 6.2 函数返回值
- [ ] 【强制】不返回多个用于控制流程的状态
- [ ] 【强制】函数返回值个数不超过3个
- [ ] 返回值含义不明确时给返回值命名

### 6.3 函数实现
- [ ] map或slice作为参数时，考虑外部修改是否符合预期

### 6.4 分组
- [ ] 函数放在struct/const/var后面
- [ ] 构造函数放在其他函数之前
- [ ] 导出函数放在非导出函数前面
- [ ] 同一文件中函数按接收者分组
- [ ] 普通工具函数在文件末尾
- [ ] 函数按粗略调用顺序排序

## 07. 语言特性

### 7.1 变量声明
- [ ] 就近声明，变量声明位置靠近使用位置
- [ ] 相似声明放在一组
- [ ] 全局变量使用var关键字，提供初始化表达式时省略类型
- [ ] 局部变量使用短变量声明（:=）
- [ ] 尽量缩小变量作用范围

### 7.2 语句
- [ ] 【强制】禁止变量隐藏
- [ ] 【强制】指针操作时判断是否为nil
- [ ] 【强制】数字运算时做好长度限制，防止溢出/反转/截断/符号错误

### 7.3 魔数
- [ ] 【强制】除0和1外不使用魔法数字
- [ ] 【强制】字符串字面量出现>=2次用常量代替

### 7.4 for语句
- [ ] 不存储循环变量的地址
- [ ] 不在循环体内修改循环变量
- [ ] for/switch和for/select使用break label

### 7.5 range语句
- [ ] 【强制】只需要key时丢弃value
- [ ] 【强制】只需要value时第一项置为下划线
- [ ] 【强制】遍历较大元素优先使用下标遍历

### 7.6 switch语句
- [ ] 【强制】必须有default
- [ ] 【强制】不在switch子句末尾使用没有目标标签的break

### 7.7 goto语句
- [ ] 【强制】业务代码禁止使用goto

### 7.8 结构体
- [ ] 【强制】结构体字段按内存占用最小原则排序
- [ ] 【强制】结构体拷贝时考虑浅拷贝问题
- [ ] 【强制】结构体初始化使用标签语法，不省略字段名
- [ ] 【强制】结构体slice和map初始化时省略重复类型字面值
- [ ] 使用&T{}代替new(T)
- [ ] 初始化时忽略零值字段
- [ ] 声明零值结构使用var
- [ ] 嵌入结构体位于顶部，用空行分隔
- [ ] 避免在公共结构体中嵌入类型
- [ ] 防止过度使用setter/getters

### 7.9 切片
- [ ] 【强制】非零值slice使用make()初始化，尽可能指定容量
- [ ] 【强制】make分配内存时，外部输入长度必须校验
- [ ] 【强制】slice索引操作时判断长度是否合法
- [ ] 【强制】禁止同时对新旧切片操作
- [ ] 【强制】copy函数确保dst长度符合预期，src和dst不重叠
- [ ] 空切片使用var声明
- [ ] 使用nil替代长度为0的slice
- [ ] 始终使用len(s) == 0检查切片是否为空
- [ ] 返回长度为零的切片时返回nil
- [ ] sort.searchXXX方法不能通过返回值>元素个数判断存在性

### 7.10 map
- [ ] 【强制】固定元素列表用map初始化列表，其他使用make
- [ ] 【强制】预估map上限时提供容量信息
- [ ] 【强制】make分配内存时，外部输入长度必须校验
- [ ] 【强制】不通过零值判断map中键是否存在，使用seconds, ok = timeZone[tz]
- [ ] 【强制】禁止遍历map时写入key
- [ ] 【强制】禁止向nil的map插入键值对
- [ ] 为map增加setter/getters
- [ ] 多层map使用类型别名定义描述性名字

### 7.11 channel
- [ ] 【强制】channel通常size为1或无缓冲，其他尺寸需注释说明
- [ ] 【强制】channel必须使用make初始化
- [ ] 【强制】注意优雅关闭channel，禁止多次关闭或向已关闭channel写入
- [ ] 【强制】禁止重复关闭channel
- [ ] 【强制】select多个channel同时有事件时会随机执行
- [ ] 尽量限制channel访问权限

### 7.12 协程
- [ ] 【强制】不泄漏goroutine，必须有等待goroutine退出的方案
- [ ] 【强制】协程池进行最大数量限制
- [ ] 【强制】不在init()使用goroutines
- [ ] 支持信号控制输出所有goroutine的stack dump信息

### 7.13 上下文
- [ ] 【强制】context始终是第一个参数
- [ ] 【强制】不传入nil的Context，不知道用哪种时使用context.TODO()
- [ ] 【强制】WithTimeout/WithDeadline/WithCancel的cancel函数最终需执行
- [ ] rpc/数据库操作等接口支持Context时应加超时设置

### 7.14 格式化
- [ ] 【强制】Printf-style函数的格式字符串在函数外声明为const常量
- [ ] 【强制】类printf函数名称以f结尾
- [ ] 【强制】格式化输出外部数据时使用%q进行安全转义

### 7.15 字符串
- [ ] 【强制】禁止反复从固定字符串创建字节Slice
- [ ] 【强制】基本数据类型和字符串转换使用strconv代替fmt
- [ ] 【强制】使用strings.Builder时禁止复制操作
- [ ] 行内字符串拼接：<=3个用+，>3个或有类型转换用fmt.Sprintf()
- [ ] 非行内字符串拼接使用strings.Join/Builder/bytes.Buffer/byte[]
- [ ] 防止无用字符串拷贝
- [ ] 需要转义时使用原始字符串字面值（`）
- [ ] 判断字符串是否为空使用str == ""
- [ ] strings.Split分割时考虑非预期的空字符串

### 7.16 初始化函数
- [ ] 【强制】尽可能避免使用init()，一个包只定义一个init
- [ ] 【强制】init()内容不依赖环境或调用方式，具有完全确定性
- [ ] 【强制】init()避免访问全局或环境状态
- [ ] 【强制】init()避免I/O操作
- [ ] 【强制】包内多个init不能有依赖关系

### 7.17 接口
- [ ] 【强制】包含nil指针的接口不是nil接口，不能用== nil判断
- [ ] 接口不需要使用指针
- [ ] 结构体实现接口时添加编译期校验
- [ ] 包之间调用使用接口而非具体实现类
- [ ] 使用接口实现多态
- [ ] 防止接口污染：不滥用interface，控制接口粒度

### 7.18 枚举
- [ ] 【强制】为枚举类型创建相应类型
- [ ] 为枚举类型添加string方法
- [ ] 让iota从a + 1开始增量

### 7.19 反射
- [ ] 防止滥用反射

### 7.20 主函数退出
- [ ] 使用os.Exit或log.Fatal*立即退出，而非panic
- [ ] 函数不容失败时使用must前缀命名

### 7.21 构建标签
- [ ] 【强制】使用build tag在编译期应对不同环境

### 7.22 Mutex
- [ ] 【强制】禁止拷贝sync.Mutex
- [ ] 【强制】mutex作为结构体非指针字段，不直接嵌入结构体
- [ ] 不使用指向mutex的指针

### 7.23 序列化
- [ ] 【强制】序列化到JSON/YAML等格式使用相关标记注释
- [ ] 【强制】序列化字段必须大写字母开头
- [ ] 【强制】反序列化时确保接受对象没有历史数据

### 7.24 容器
- [ ] 【强制】容器程序使用go.uber.org/automaxprocs/maxprocs设置GOMAXPROCS

### 7.25 许可证
- [ ] 【强制】禁止引入GPL许可证的第三方包

### 7.26 time
- [ ] 【强制】定时器使用time.NewTicker()和defer timer.Stop()，禁用time.After()
- [ ] 【强制】使用time包处理时间

### 7.27 原子操作
- [ ] 【强制】使用go.uber.org/atomic包代替sync/atomic包

### 7.28 随机数
- [ ] 【强制】使用crypto/rand's Reader代替math/rand

### 7.29 unsafe包
- [ ] 【强制】禁止使用unsafe包，cgo场景除外

### 7.30 cgo
- [ ] 【强制】不把Go中分配的内存地址传递给C使用
- [ ] 【强制】C中使用thread local变量时使用runtime.LockOSThread和UnlockOSThread
- [ ] 多次调用C函数时尽量减少调用次数

## 08. 国际化

### 8.1 标点符号
- [ ] 【强制】翻译中英文使用英文标点，中文使用中文标点
- [ ] 【强制】分隔同类对象时，中文用顿号，英文用逗号

### 8.2 翻译
- [ ] 【强制】所有需要翻译的字符串都用T()
- [ ] 【强制】翻译词条不应为格式化后的字符串

## 使用说明

1. **编码前**: 浏览相关章节，熟悉规范要求
2. **编码中**: 随时参考对应的规范条目
3. **编码后**: 逐项检查所有【强制】标记的条目
4. **代码审查**: 使用本checklist进行系统性审查
5. **持续改进**: 定期回顾，养成良好编码习惯

**优先级说明**:
- 【强制】: 必须遵守，违反会导致代码审查不通过
- 无标记: 建议遵守，有助于提高代码质量

**工具配置**:
- 配置IDE使用gofmt/goimports自动格式化
- 配置git pre-commit hook运行golangci-lint
- 配置CI/CD流程自动运行所有检查
