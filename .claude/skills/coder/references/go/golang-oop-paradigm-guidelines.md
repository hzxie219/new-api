# Golang 面向对象编程规范

## 01. 设计原则

- [ ] **单一职责**: 每个结构体只负责一项职责
- [ ] **开闭原则**: 对扩展开放，对修改关闭
- [ ] **接口隔离**: 使用多个专门的接口，而非单一通用接口
- [ ] **依赖倒置**: 依赖于接口，而非具体实现
- [ ] **组合优于继承**: 通过嵌入和组合复用代码

---

## 02. 结构体设计

- [ ] 使用私有字段，通过公共方法访问
- [ ] 提供构造函数 `NewXxx` 创建实例
- [ ] 避免直接暴露可变的内部状态

---

## 03. 方法接收者

- [ ] **值接收者**: 用于小型、不可变的结构体
- [ ] **指针接收者**: 用于需要修改状态或大型结构体
- [ ] **一致性**: 同一类型的所有方法使用相同类型的接收者

---

## 04. 接口设计

- [ ] 小接口优于大接口
- [ ] 接口由使用方定义
- [ ] 接受接口，返回具体类型

---

## 05. Functional Options 模式

适用场景：构造函数有多个可选参数、需要默认值、参数可能增长

- [ ] Option 类型定义为 `type Option func(*T) error`
- [ ] Option 函数以 `With` 前缀命名
- [ ] 在 Option 函数中进行参数验证
- [ ] 构造函数提供合理的默认值
- [ ] 构造函数返回 `(*T, error)` 支持错误处理

---

## 06. 依赖注入

- [ ] 通过构造函数注入依赖
- [ ] 依赖使用接口类型
- [ ] 提供默认实现以简化使用
