# Python 编码规范 Checklist

## 1. 风格规范 (Style)

### 1.1 标识符命名 (Label)
- [ ] **模块与包命名**：全小写，模块可用下划线，使用名词
- [ ] **类命名**：CapWords格式，首字母大写，使用名词
- [ ] **异常命名**：CapWords + Error后缀
- [ ] **函数与方法**：动宾结构，全小写，下划线分隔，不少于3个字符
- [ ] **变量命名**：全小写加下划线，避免单字符（除计数器i/j/k、异常e、文件句柄f）
- [ ] **常量命名**：全大写，可用下划线
- [ ] **文件名**：.py后缀，不含连字符(-)
- [ ] **类型别名**：CapWords方式
- [ ] **类型变量**：CapWords，短名称（T, AnyStr），协变/逆变加_co/_contra后缀
- [ ] **避免隐藏内置名字**
- [ ] **命名方式统一**：同一文件内保持一致
- [ ] **类属性冲突**：与关键字冲突时后缀下划线
- [ ] **私有属性**：前缀双下划线避免子类冲突
- [ ] **方法参数**：实例方法第一参数self，类方法第一参数cls

### 1.2 缩进 (Indent)
- [ ] 使用4个空格缩进
- [ ] 禁止使用Tab
- [ ] 禁止混用Tab和空格

### 1.3 空行 (Blank)
- [ ] 类和顶级函数之间：2个空行
- [ ] 类内方法之间：1个空行
- [ ] 函数内逻辑段落间：1个空行
- [ ] 控制语句(if/while/for)后：1个空行

### 1.4 折行 (Wrap)
- [ ] 使用括号隐式续行，避免反斜杠
- [ ] 垂直隐式连接和悬挂式缩进
- [ ] 第一行不放参数
- [ ] 右括号可在表达式结尾或另起一行
- [ ] 低优先级操作符处拆分，操作符放新行首

### 1.5 括号 (Parenthesis)
- [ ] 宁缺毋滥
- [ ] return语句和条件语句中避免不必要括号

### 1.6 注释 (Comment)
- [ ] 模块头：文档字符串说明用途和注意事项
- [ ] 函数头：说明功能、参数、返回值
- [ ] 类头：说明类功能
- [ ] 文件头：功能、作者、修改者信息
- [ ] 行注释：在行上方，不在行尾
- [ ] if分支：注释条件和功能
- [ ] 循环：注释循环条件和功能
- [ ] 优先使用中文注释

### 1.7 空格 (Space)
- [ ] 右括号前不加空格
- [ ] 逗号、冒号、分号前不加空格
- [ ] 函数左括号前不加空格
- [ ] 序列左括号前不加空格
- [ ] 操作符左右各一个空格
- [ ] 函数默认参数赋值符左右不加空格
- [ ] 函数注解：冒号后加空格，箭头前后加空格

### 1.8 尾随逗号 (Trailing Comma)
- [ ] 仅当括号与最后元素不同行时添加

### 1.9 编码格式 (Coding Format)
- [ ] UTF-8无BOM编码
- [ ] 添加 `# coding=utf-8`
- [ ] 以 `#!/usr/bin/env python<version>` 开头
- [ ] Unix格式换行符
- [ ] 禁止使用 `sys.setdefaultencoding`

## 2. 错误处理 (Exception)

### 2.1 异常抛出 (Raise)
- [ ] 外部数据添加try/except处理
- [ ] 使用 `raise ValueError('message')` 格式
- [ ] except/finally中避免抛新异常
- [ ] Python3用异常链：`raise X from Y`

### 2.2 异常捕获 (Catch)
- [ ] 避免裸except或捕获Exception
- [ ] 同级try/catch捕获
- [ ] 多异常用 `except (Ex1, Ex2):`

### 2.3 异常处理 (Exception)
- [ ] 使用基于对象的异常
- [ ] 定义模块异常基类继承Exception
- [ ] finally中放清理代码
- [ ] finally中禁止return/break/continue
- [ ] 禁止pass忽略异常
- [ ] 最小化try块代码量

### 2.4 异常日志 (Log)
- [ ] LOG.exception仅用于except作用域
- [ ] 用LOG.exception打印堆栈，不用LOG.error
- [ ] 打印异常条件和相关变量

### 2.5 断言 (Assert)
- [ ] 仅用于内部正确性
- [ ] 禁止在断言中调用有副作用的函数

## 3. 函数/方法 (Function)

### 3.1 默认参数 (Default Parameter)
- [ ] 禁止动态默认值如 `now=time.time()`
- [ ] 禁止可变对象作默认值（列表、字典）
- [ ] 默认参数不调用业务函数
- [ ] 新参数考虑向后兼容

### 3.2 可变参数 (Var Parameter)
- [ ] 尽量避免 `*args, **kwargs`

### 3.3 命令行参数 (Cmd Parameter)
- [ ] 值不用JSON格式

### 3.4 参数传递 (Parameter)
- [ ] 禁止用locals()、globals()传参

### 3.5 返回值 (Return)
- [ ] 最多3个返回值
- [ ] 多于3个用具名形式（class/namedtuple/dict）
- [ ] 异常代替错误返回值
- [ ] 保持返回值类型一致

### 3.6 特性 (Properties)
- [ ] 用@property创建特性
- [ ] 子类不覆写父类特性
- [ ] 特性包含简单计算逻辑

### 3.7 类型注解 (Type Annotation)
- [ ] Python3代码添加类型注解
- [ ] 使用pytype等工具检查

### 3.8 嵌套函数 (Internal Function)
- [ ] 避免嵌套/局部/内部类或函数

### 3.9 词法作用域 (Lexical Scope)
- [ ] 闭包中禁止直接变量绑定
- [ ] 嵌套函数只读访问外层变量

### 3.10 Lambda函数
- [ ] 用operator模块代替简单lambda
- [ ] 限制60-80字符内

### 3.11 装饰器 (Decorator)
- [ ] 使用functools.wraps装饰返回函数
- [ ] 避免外部依赖
- [ ] 不调用业务逻辑

## 4. 语言特性 (Language)

### 4.1 导入 (Import)
- [ ] 放在文件顶部
- [ ] 禁止 `import *`
- [ ] 每行一个导入
- [ ] 不导入未使用模块
- [ ] 使用绝对导入
- [ ] 按序排列：__future__、标准库、第三方库、自己的库

### 4.2 主程序 (Main)
- [ ] 检查 `if __name__ == '__main__'`
- [ ] 非__main__文件禁止直接执行业务函数

### 4.3 特殊变量 (Dunders)
- [ ] `__all__`等放在模块文档后、导入前

### 4.4 __init__.py
- [ ] 保持为空文件
- [ ] 禁止业务逻辑

### 4.5 语句行 (Statement)
- [ ] 链式赋值仅用于简单变量
- [ ] 不在同一行写多条语句
- [ ] if/for/while执行语句另起一行

### 4.6 If语句
- [ ] 过长if else用传统格式
- [ ] 判断语句不超一行
- [ ] 使用all/any处理多条件
- [ ] 代码块不超10行
- [ ] 短分支优先
- [ ] 短路原则，提前返回

### 4.7 列表推导 (Comprehensions)
- [ ] 简单场景使用
- [ ] 禁止多层loop或filter

### 4.8 迭代器 (Iterator)
- [ ] 只读遍历用内置迭代
- [ ] 用in/not in判断存在

### 4.9 生成器 (Generator)
- [ ] 长列表数据用yield
- [ ] Python2用xrange代替range

### 4.10 条件表达式 (Cond Expression)
- [ ] 仅一行内使用
- [ ] 禁止嵌套

### 4.11 布尔值 (Bool)
- [ ] 使用隐式false
- [ ] 不用==比较False
- [ ] 序列用 `if not seq:` 判空
- [ ] NumPy数组用.size属性

### 4.12 单例对象比较
- [ ] 用is/is not比较None
- [ ] 不用 `not ... is`

### 4.13 过时特性 (Obsolete)
- [ ] 用字符串方法代替字符串模块
- [ ] 用函数调用代替apply()

### 4.14 威力过大特性
- [ ] 禁止元类、字节码访问、动态编译、动态继承、对象父类重定义、导入黑客、反射、系统内修改

### 4.15 字符串 (String)
- [ ] 多行字符串用三重双引号
- [ ] 多字符串拼接用join
- [ ] 用startswith/endswith检测前后缀
- [ ] 用f-string或format格式化
- [ ] 日志用字符串字面量+占位符
- [ ] 正则代替isalpha判断
- [ ] 循环中避免+=累加字符串

### 4.16 字典 (Dict)
- [ ] 确定存在key时用dict['key']
- [ ] get显式声明默认值
- [ ] 用JSON保存字典
- [ ] 单字段用索引，多字段用update
- [ ] 不假定字典有序
- [ ] 用 `for x in dict` 遍历
- [ ] setdefault设置默认值

### 4.17 元组 (Tuple)
- [ ] 单元素元组加逗号：`(1,)`

### 4.18 全局变量 (Global)
- [ ] 禁止全局变量，用类静态变量
- [ ] 例外：默认配置、模块常量、缓存

### 4.19 变量初始化 (Init)
- [ ] 全局变量需初始化
- [ ] 类成员在构造函数初始化
- [ ] 局部变量使用前初始化

### 4.20 列表删除
- [ ] 遍历时禁止删除元素

### 4.21 性能 (Performance)
- [ ] 适用多种Python实现
- [ ] 可变对象用+=
- [ ] 复杂匹配用re库
- [ ] 内部数据传递用数据类
- [ ] 大数据in运算避免列表

### 4.22 排序 (Sort)
- [ ] 复杂比较实现全部六个操作

### 4.23 对象比较
- [ ] 用isinstance()比较类型

### 4.24 整数比较
- [ ] 禁止用is比较整数

### 4.25 魔数 (Magic)
- [ ] 用常量代替魔数

### 4.26 资源管理 (Resource)
- [ ] with语句管理文件
- [ ] 显式关闭资源
- [ ] 大量实例用__slots__

### 4.27 时间 (Time)
- [ ] 用dateutil.tz.gettz()获取时区
- [ ] UTC用calendar.timegm()转时间戳

### 4.28 Protobuf
- [ ] 避免Python关键字

### 4.29 哈希 (Hash)
- [ ] 持久化数据禁用内置hash

### 4.30 类成员变量
- [ ] 禁止通过实例操作类变量
- [ ] 初始化函数禁止业务逻辑

### 4.31 外部程序 (Popen)
- [ ] subprocess加close_fds=True
- [ ] 重定向stderr

### 4.32 UUID
- [ ] 比较时转string类型

### 4.33 反序列化 (Deserialize)
- [ ] 用yaml.safe_load
- [ ] 禁止pickle模块

## 5. 并发 (Concurrent)

### 5.1 线程/协程
- [ ] 父任务结束时回收协程
- [ ] 协程内禁止阻塞函数
- [ ] CPU密集型禁止多线程
- [ ] Python2用tornado/twisted，Python3用asyncio

### 5.2 进程 (Process)
- [ ] 大对象频繁更新禁用multiprocessing

### 5.3 文件刷新 (Flush)
- [ ] 并发读写及时flush或关闭

## 6. 国际化 (i18n)

### 6.1 标点符号
- [ ] 英文用英文标点
- [ ] 中文用中文标点
- [ ] 分隔对象：中文顿号，英文逗号

### 6.2 翻译 (Translate)
- [ ] 需翻译字符串用_()
- [ ] 不翻译格式化后字符串
- [ ] 用命名占位符%(name)s

### 6.3 字符串编码
- [ ] 非ASCII返回UTF-8
- [ ] 禁止代码含非ASCII
- [ ] 仅用ASCII或UTF-8编码

## 7. 安全特性 (Security)

### 7.1 文件权限
- [ ] 避免过高权限（如777）

### 7.2 硬编码
- [ ] 禁止硬编码0.0.0.0绑定
- [ ] 禁止硬编码密码
- [ ] 禁止默认参数硬编码密码

### 7.3 临时文件
- [ ] 使用tempfile
- [ ] 及时删除临时文件
- [ ] 避免/tmp等公共目录

### 7.4 用户权限
- [ ] 避免root权限执行
- [ ] run_as_root设为False

### 7.5 命令注入
- [ ] 用ast.literal_eval代替eval
- [ ] 用raw_input代替input（Python2）
- [ ] 充分校验不安全函数输入
- [ ] 禁止Linux命令通配符

### 7.6 加密算法
- [ ] 用os.urandom或SystemRandom
- [ ] DSA密钥>=2048位

### 7.7 XML库
- [ ] 使用defusedxml库

### 7.8 SSL
- [ ] requests设verify=True
- [ ] 使用高版本SSL协议（TLSv1.2+）

### 7.9 SQL注入
- [ ] 禁止SQL字符串拼接

### 7.10 框架安全
- [ ] Flask：autoescape=True
- [ ] Flask：禁止debug=True
- [ ] Django：禁用make_safe

### 7.11 私有属性
- [ ] 私有变量/函数前缀双下划线

### 8.1 检查要求
- [ ] 改动代码遵守规范

## 使用说明

1. **强制规范**：必须遵守的规则，违反会导致代码质量问题
2. **建议规范**：推荐遵守的最佳实践，提高代码质量
3. **检查方式**：
   - 编码前：快速浏览相关章节
   - 编码中：参考具体条目
   - 编码后：逐项检查确认
   - Code Review：作为评审标准

## 优先级建议

### 高优先级（必须遵守）
- 命名规范
- 缩进和格式
- 异常处理
- 安全特性
- 资源管理

### 中优先级（强烈建议）
- 函数设计
- 代码组织
- 性能优化
- 类型注解

### 低优先级（建议遵守）
- 注释规范
- 代码风格统一
- 工具检查
