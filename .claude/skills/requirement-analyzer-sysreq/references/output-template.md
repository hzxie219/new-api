## 模板使用说明

1. **替换所有占位符**：将所有 `{}` 标记的占位符替换为分析得出的实际内容
2. **严格区分需求类型**：
   - 第2章只写功能性需求（业务功能、接口、数据处理）
   - 第5章只写非功能性需求（性能、安全、可靠性等）
   - 🚨 绝对禁止将性能指标、安全要求、监测等非功能需求放入第2章
3. **删除不适用的章节**：如没有配置文件变更，删除3.2章节
4. **保持严格格式**：维护表格和代码块的格式
5. **使用中文**：所有描述性文本使用中文，技术术语可保留英文
6. **包含行号**：所有代码引用包含行号，格式为 `项目名/路径/文件.py:行号`
7. **标记功能类型**：功能标题一致使用 "(增量功能)" 或 "(全新功能)" 后缀
8. **分离接口和后台功能**：在2.2章节下将接口功能和后台功能分为不同子章节
9. **使用表格或列表**：新增接口的输入/输出用表格，增量接口的变更用列表
10. **省略后台功能的输入/输出**：后台功能完全省略"输入定义"和"输出定义"章节

## 输出前质量检查

- [ ] 所有代码引用指向真实文件（无虚构路径）
- [ ] 接口路径从实际代码装饰器中提取
- [ ] 功能分类完整且准确
- [ ] 所有功能都有设计需求标记（"需要方案设计"、"需要方案选型"）
- [ ] **第2章只包含功能性需求，无性能/安全等非功能需求**
- [ ] **第5章包含所有非功能性需求，从系统需求文档中正确提取**
- [ ] 文档结构完全遵循模板
- [ ] 描述性文本使用中文，技术术语使用英文
- [ ] 输出路径正确：`doc/requirement/requirement.md`
