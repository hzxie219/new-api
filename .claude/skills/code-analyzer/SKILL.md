# code-analyzer

提供代码分析能力,用于检测 bug、安全问题和项目规范违规。

## 功能说明

提供以下代码分析能力:

| 功能分类 | 说明 |
|----------|------|
| Bug 扫描 | 检测明显的运行时错误和逻辑问题 |
| 安全分析 | 识别安全漏洞和敏感信息泄露 |
| 规范检查 | 验证代码是否符合项目知识库规范 |
| Diff 分析 | 分析代码变更的影响范围 |

## 操作指南

### 1. 获取代码 Diff

```bash
# 获取变更文件列表
git diff --name-only origin/<target>...origin/<source>

# 获取详细 diff
git diff origin/<target>...origin/<source> -- <文件列表>
```

### 2. 识别代码文件

```bash
# 检查文件类型
git show origin/<branch>:"$file" | file -

# 统计文件行数
git show origin/<branch>:"$file" | wc -l
```

### 3. Bug 扫描规则

扫描以下类型的问题:

| 问题类型 | 检测规则 |
|----------|----------|
| 空指针引用 | 变量使用前未检查null/undefined |
| 资源泄露 | 打开的资源未正确关闭 |
| 类型错误 | 类型不匹配的操作 |
| 边界错误 | 数组越界、整数溢出 |
| 逻辑错误 | 条件判断错误、死循环 |

### 4. 安全分析规则

检测以下安全问题:

| 问题类型 | 检测规则 |
|----------|----------|
| 硬编码凭据 | 代码中包含密钥、密码、Token |
| SQL注入 | 未参数化的SQL查询 |
| XSS漏洞 | 未转义的用户输入输出 |
| 命令注入 | 未过滤的shell命令拼接 |
| 路径遍历 | 未验证的文件路径操作 |

### 5. 规范检查

根据 `doc/kb/` 目录下的知识库文档验证:

1. 加载项目知识库文档
2. 提取规范要求
3. 对照代码变更检查违规
4. 引用具体规则条目

## 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| Diff获取失败 | 分支不存在 | 先fetch远程分支 |
| 文件过大 | 单文件超过500行 | 跳过大文件,记录原因 |
| 二进制文件 | 非文本文件 | 跳过二进制文件 |
| 知识库缺失 | doc/kb目录不存在 | 跳过规范检查 |
