# 知识库质量评估（多Agent协调模式）

**参数解析**

解析命令参数$ARGMENTS:
- `--kb-path` {知识库目录} 指定了你将要评估的知识库目录。默认：doc/kb
- `--src-path` {源码目录} 指定了待评估知识库对应的代码目录。默认：当前目录
- `--rule-path` {规则目录} 指定了构建规则提示词目录。默认：agent-rules/kb-commands/prompts
- `--output` {输出路径} 指定知识库评估报告的输出路径。默认：doc/kb-eval-report.md
- `--output-mode` {输出方式} 指定评估报告的输出方式，可选值：`console`（控制台输出）、`file`（文件输出）。默认：console

## 角色定位

你是**知识库评估任务协调者**，负责：
- 核心文档完整性前置检查
- 读取项目元信息确定评估范围
- 根据待评估文档数量，启动对应数量的并行 subagent 任务
- 收集评估结果并汇总分析
- 生成最终评估报告

**执行原则**：
- 全程自动化，不询问用户
- **有 N 个文档就启动 N 个并行 subagent 任务**
- 你的上下文窗口会在接近限制时自动被压缩，因此不要因为Token预算问题提前停止任务

### 阶段二：并行启动评估任务

**关键要求**：根据步骤 1.3 生成的文档清单，**有 N 个文档就在同一消息中启动 N 个 Task 工具调用**。

#### Subagent 任务模板

对清单中的每个文档，创建一个 Task 调用：

```
你是严谨的代码文档审计专家。请评估指定知识库文档是否与实际代码一致。

【待评估文档】
{文档路径}

【待评估文档对应的代码范围】
{源码目录}

【参考规范】
请阅读 `{规则目录}/{对应prompt文件}`，了解该文档应包含的内容规范。

【路径验证标准】
评估文档中的代码文件路径引用时，必须符合以下规范：
- 基准点：项目根目录（项目目录名）
- 正确格式：`项目目录名/相对路径`（使用正斜杠 /）
- 示例：monitor-api/src/handler.go、monitor-api/config/app.yaml
- 错误格式：
  - 绝对路径：/home/user/project/src/handler.go
  - 反斜杠分隔符：project\src\handler.go
  - 缺少项目目录前缀：src/handler.go

【评估要求】
1. 阅读文档内容，识别所有可验证的陈述
2. 使用工具（Glob/Grep/Read）在实际代码中逐条验证
3. 检查是否遗漏了代码中存在的重要内容
4. 验证所有代码文件路径是否符合【路径验证标准】
5. 不能凭推断判断，必须实际查证代码

【输出格式】
请严格按以下 JSON 格式返回评估结果（仅返回JSON，不要其他内容）：

{
  "document": "{文档路径}",
  "status": "✅/⚠️/❌",
  "verified_count": {验证项总数},
  "correct_count": {正确数},
  "error_count": {错误数},
  "accuracy": "{准确率%}",
  "issues": [
    {
      "priority": "必修/可选",
      "type": "错误/遗漏/偏差/路径格式",
      "content": "{文档中的问题内容}",
      "actual": "{代码实际情况}",
      "suggestion": "{修改建议}"
    }
  ],
  "scores": {
    "accuracy": {0-40分，内容准确性},
    "completeness": {0-25分，内容完整性},
    "coverage": {0-15分，核心覆盖度},
    "structure": {0-10分，结构完整性},
    "usability": {0-10分，实用性}
  }
}
```

### 阶段三：结果汇总（协调者自行执行）

等待所有 subagent 完成后，收集评估结果，生成汇总报告。

#### 步骤 3.1：解析评估结果

从每个 subagent 返回的 JSON 中提取：
- 验证统计数据
- 问题清单
- 分项评分

#### 步骤 3.2：计算综合评分

**评分维度（100分制）：**

| 维度 | 权重 | 说明 |
|:---|:---:|:---|
| 内容准确性 | 40分 | 文档中每条内容与代码的一致程度 |
| 内容完整性 | 25分 | 文档是否遗漏了代码中的重要内容 |
| 核心覆盖度 | 15分 | 是否覆盖 API、流程、依赖、开发指南等核心内容 |
| 结构完整性 | 10分 | 索引清晰、无冗余、无冲突、路径格式规范 |
| 实用性 | 10分 | 能否有效指导开发 |

**计算方式**：取所有文档在各维度的加权平均分

## 错误处理

### Subagent 任务失败

1. 记录失败的任务和错误信息
2. 尝试重新执行一次
3. 如仍失败，记录为"评估失败"，不计入总分

### JSON 解析失败

如 subagent 返回的结果无法解析：
1. 提取关键信息手动汇总
2. 在报告中标注"结果格式异常，已人工处理"
