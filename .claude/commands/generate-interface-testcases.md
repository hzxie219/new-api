# 接口测试用例生成器

## 任务描述

基于TDD流程，从需求设计文档自动生成全面覆盖的接口测试用例文档，包括正常场景、异常场景、边界条件和安全测试。

### 全局约束

1. **TDD方法论**: 先分析需求，再设计测试
2. **基于真实需求**: 所有测试用例必须源自概设文档
3. **100%接口覆盖**: 识别并覆盖所有API接口
4. **参数完整性**: 接口定义的每个参数必须在测试用例中体现
5. **标准化输出**: 输出符合规范的测试用例文档

### 处理步骤

#### 步骤1: 需求分析与接口识别
调用SKILL /api-test-analyzer
- 输入: 概设文档
- 输出: 结构化接口清单(JSON)、业务规则总结
- 重点: API定义、参数约束、错误处理、安全要求

#### 步骤2: 测试用例设计与文档生成
调用SKILL /testcase-designer
- 输入: 步骤1的接口清单
- 输出: 完整的测试用例文档(Markdown)
- 内容:
  - 测试概述、接口清单
  - 详细测试用例表格
  - 参数覆盖矩阵
  - 覆盖度统计、质量评估

## 注意事项

1. **按优先级排序**: P0 > P1 > P2
2. **场景分类**: 正常/异常/边界/安全
3. **独立可执行**: 每个用例独立运行
4. **参数覆盖矩阵**: 必须包含，确保无遗漏
