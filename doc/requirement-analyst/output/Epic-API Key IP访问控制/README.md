# 【Epic】API Key IP 访问控制

**价值**
1. 降低 API Key 盗用后的损失风险，支持按来源 IP 精准管控 Key 的可用范围，将事故处置时间从"人工改密钥"缩短为秒级策略生效
2. 支持企业级用户对 Key 来源 IP 的精细化管控诉求，引入 CIDR 网段支持，解决大型企业出口 IP 段逐条填写的痛点
3. 提升平台安全信誉，满足对内部系统发放专属 Key、对异常来源快速封堵的双重场景诉求

**目标用户**

| 用户角色 | 使用场景 |
|---------|---------|
| 平台管理员 | 为敏感 Key 配置严格 IP 白名单，防止 Key 外泄后被滥用 |
| 普通用户 / 开发者 | 为自己的 API Key 绑定来源 IP 段，提升账户安全性 |
| 平台运营者 | 在事故处置时快速黑名单封堵特定攻击 IP 段 |

**背景和现状**
1. 当前 API Key 的 IP 白名单仅支持精确 IP 匹配，不支持 CIDR 网段，企业出口 IP 段须逐条填写，运维成本高
2. 只有白名单模式（允许指定来源），无黑名单模式（阻断特定来源），在事故处置场景下无法快速封堵攻击来源
3. 在反向代理 / CDN 场景下，直接读取 `RemoteAddr` 得到的是代理 IP 而非真实客户端 IP，导致现有 IP 策略在代理场景下完全失效；同时无条件信任 `X-Forwarded-For` 则被攻击者伪造，造成策略绕过

**特性规划**
1. Feature 1 — IP 策略配置管理：提供统一的 `PUT /api/keys/{id}/ip_policy` 接口，支持白名单/黑名单两种模式，IP 列表支持精确 IP 和 CIDR 网段
2. Feature 2 — IP 访问策略执行：实现可信代理感知的真实 IP 提取机制，在鉴权中间件中执行白名单/黑名单校验，违规请求返回标准化 403 响应

**成效指标（定性 / 定量）**
1. 【定量】IP 策略配置接口响应时间 P99 < 200ms
2. 【定量】IP 策略校验在鉴权流程中增加的延迟 < 1ms（单请求 CIDR 匹配，O(n)，n ≤ 100）
3. 【定性】在反向代理场景下，IP 策略可正确识别并拦截伪造 XFF 头的攻击请求
4. 【定性】支持同时运行 SQLite / MySQL / PostgreSQL 三种数据库，新增 `ip_policy` 字段无需手动迁移

**风险与依赖**
1. **遗留字段共存风险**：`tokens` 表原有 IP 白名单字段（如有）与新 `ip_policy` 字段并存，需明确是否废弃旧字段，避免双重校验产生语义歧义
2. **Q1 待确认**：`ips=[]` 且 `mode` 非空时语义（拒绝所有 vs 等同清空策略），优先级高，影响用户理解和后端实现
3. **Q4 待确认**：普通用户是否可为自己的 Key 设置黑名单模式，影响接口权限设计
4. **代理配置风险**：`TRUSTED_PROXIES` 默认值设置不当（如默认信任所有），可能导致 IP 策略被攻击者绕过，需在部署文档中明确警示
