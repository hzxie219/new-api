## 【Feature 2】IP 访问策略执行

### 场景描述

每次使用 API Key 调用业务接口时，网关需要：
1. 正确识别请求的真实来源 IP（区分直连请求与经由可信代理转发的请求）
2. 根据该 Key 绑定的 IP 策略（白名单/黑名单）判断此 IP 是否允许访问
3. 对违规请求立即返回 HTTP 403，附带固定错误码 `IP_NOT_ALLOWED`

### 用户痛点

- 在 Nginx/CDN 反向代理场景下，直接读取 `RemoteAddr` 得到的是代理 IP 而非真实客户端 IP，导致 IP 策略完全失效
- 但无条件信任 `X-Forwarded-For` 头则会被攻击者伪造，造成 IP 策略被绕过
- 现有错误响应格式不统一，调用方无法通过错误码程序化识别"IP 被拒绝"与其他 403 错误

### 解决方案

引入可信代理（Trusted Proxy）配置：只有请求来自可信代理时才读取 `X-Forwarded-For` 中的客户端 IP，否则使用直连远端 IP。校验逻辑在鉴权中间件中统一执行，违规返回标准化错误响应。

### 包含 Story

- Story 1.2.1: 请求鉴权时获取真实客户端 IP
- Story 1.2.2: 按 IP 策略拦截非法来源请求
