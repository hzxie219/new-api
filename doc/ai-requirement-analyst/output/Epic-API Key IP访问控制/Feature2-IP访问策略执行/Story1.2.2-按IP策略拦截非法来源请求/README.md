### 【Story 1.2.2】按 IP 策略拦截非法来源请求

#### 用户故事说明

作为 **平台网关**，我需要在每次 API Key 鉴权时，根据该 Key 绑定的 IP 策略（白名单或黑名单）对请求来源 IP 进行校验，对不符合策略的请求立即拒绝并返回标准化错误，以确保 Key 的访问范围严格受控。

#### 简化验收条件（A/C）

##### 【正常场景】白名单模式 — 来源 IP 在白名单内，请求通过

- **Given**：API Key 的 IP 策略为 `whitelist`，IP 列表为 `["1.2.3.4/32"]`；当前请求的真实客户端 IP 为 `1.2.3.4`
- **When**：使用该 API Key 调用任意业务接口
- **Then**：IP 校验通过，请求继续处理，正常返回业务响应

##### 【正常场景】黑名单模式 — 来源 IP 不在黑名单内，请求通过

- **Given**：API Key 的 IP 策略为 `blacklist`，IP 列表为 `["5.5.5.0/24"]`；当前请求的真实客户端 IP 为 `8.8.8.8`
- **When**：使用该 API Key 调用任意业务接口
- **Then**：IP 校验通过（未命中黑名单），请求继续处理

##### 【主要异常场景】白名单模式 — 来源 IP 不在白名单内，请求被拒绝

- **Given**：API Key 的 IP 策略为 `whitelist`，IP 列表为 `["1.2.3.4/32"]`；当前请求的真实客户端 IP 为 `8.8.8.8`
- **When**：使用该 API Key 调用任意业务接口
- **Then**：接口返回 HTTP 403，响应体包含错误码 `IP_NOT_ALLOWED`，请求不被转发到上游

##### 【主要异常场景】黑名单模式 — 来源 IP 命中黑名单，请求被拒绝

- **Given**：API Key 的 IP 策略为 `blacklist`，IP 列表为 `["5.5.5.0/24"]`；当前请求的真实客户端 IP 为 `5.5.5.100`
- **When**：使用该 API Key 调用任意业务接口
- **Then**：接口返回 HTTP 403，响应体包含错误码 `IP_NOT_ALLOWED`，请求不被转发到上游

#### 备注说明

- 该 Story 依赖 Story 1.2.1 提供的真实客户端 IP
- IP 策略为空（未配置）时，跳过 IP 校验，允许所有来源
- CIDR 匹配需支持 IPv4，IPv6 支持为可选项
- 错误响应格式应与现有鉴权错误保持一致（`{"success":false,"message":"...","error_code":"IP_NOT_ALLOWED"}`）
- 🔴 **重点细化**：IP 策略校验在鉴权中间件的哪个阶段执行？与现有 Token 校验（额度、有效期、IP 白名单）的顺序关系？
- 🟡 **待确认**：是否记录 IP 被拒绝的事件日志？（建议：写入数据库错误日志，type=5，便于审计）
- 🟢 **已明确**：拒绝时 HTTP 状态码为 403，错误码固定为 `IP_NOT_ALLOWED`
