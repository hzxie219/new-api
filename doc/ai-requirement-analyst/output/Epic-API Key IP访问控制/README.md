# 【Epic】API Key IP 访问控制

## 背景与价值

当前 API Key 的 IP 白名单仅支持精确 IP 匹配，缺乏 CIDR 网段支持，且只有白名单模式（允许指定来源），无法实现黑名单拦截（阻断特定来源）。

在 API Key 遭盗用或业务遭到异常来源攻击的场景下，单纯依赖 RPM 限流无法精准阻止"来源异常"的请求。运营者需要一种按来源 IP 精准控制 Key 可用范围的机制：
- 对内部系统发放的 Key，可白名单限定只有自家服务器 IP 才能使用
- 对疑似盗用或受攻击的 Key，可临时黑名单封堵特定 IP 段

同时，在反向代理/CDN 场景下，获取客户端真实 IP 需要可信代理感知，防止攻击者伪造 `X-Forwarded-For` 头绕过 IP 策略。

## 目标用户

- **平台管理员**：为敏感 Key 配置严格的 IP 白名单，防止 Key 外泄后被滥用
- **普通用户/开发者**：为自己的 API Key 绑定来源 IP 段，提升账户安全性
- **平台运营者**：在事故处置时快速黑名单封堵特定攻击 IP 段

## 预期价值

- 降低 API Key 盗用后的损失风险，提升平台安全信誉
- 支持企业级用户对 Key 来源 IP 的精细化管控诉求
- 封堵特定 IP 段后，可将事故处置时间从"人工改密钥"缩短为"秒级策略生效"

## 包含 Feature

- Feature 1: IP 策略配置管理
- Feature 2: IP 访问策略执行
