## 【Feature 1】IP 策略配置管理

### 场景描述

管理员或用户需要为指定 API Key 设置 IP 访问策略。策略由两个要素组成：
- **模式**（mode）：`whitelist`（白名单，只允许命中 IP）或 `blacklist`（黑名单，拒绝命中 IP）
- **IP 列表**（ips）：支持精确 IP（如 `1.2.3.4`）和 CIDR 网段（如 `10.0.0.0/8`）

用户通过管理后台或 API 调用 `PUT /api/keys/{id}/ip_policy` 完成策略配置。策略保存后在该 Key 的后续所有请求中立即生效。

### 用户痛点

- 现有 IP 白名单不支持 CIDR 网段，大型企业出口 IP 段需要逐条填写
- 没有黑名单模式，遭受攻击时只能改密钥而无法快速封堵来源 IP
- 策略字段散落，缺乏统一入口，配置体验不一致

### 解决方案

提供统一的 IP 策略配置接口，支持白名单和黑名单两种模式，IP 列表支持 CIDR 写法，用一次 API 调用完成策略的增删改。
