## 6. 性能需求分析

| 性能点 | 分析 | 建议 |
|--------|------|------|
| CIDR 匹配耗时 | O(n)，n 为 IP 策略条目数，单请求耗时可忽略 | 限制单 Key 最多 100 条 IP 策略 |
| CIDR 列表解析 | `ParseCIDRList` 每次调用都解析，高频 Key 有重复解析开销 | 对 Token 的已解析 CIDR 做短时间内存缓存（如 TTL=60s） |
| 可信代理校验 | 全局预编译，O(m)，m 为可信代理数量，通常 < 10，耗时极低 | 无需优化 |
| 数据库读取 | `ip_policy` 字段随 Token 一同读出，无额外查询 | 符合现有 Token 缓存架构 |
