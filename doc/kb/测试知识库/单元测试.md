# 单元测试文档

> 本文档从代码仓库中提取，说明如何编写和运行单元测试。

## 测试框架

| 项目 | 值 |
|-----|---|
| **主框架** | Go 标准库 `testing` |
| **断言库** | `github.com/stretchr/testify`（`assert` + `require` 子包） |
| **数据库（测试用）** | `github.com/glebarez/sqlite`（内存 SQLite，用于集成测试） |
| **JSON 断言辅助** | `github.com/tidwall/gjson`（用于 JSON 字段断言） |
| **HTTP 测试辅助** | Go 标准库 `net/http/httptest` + `github.com/gin-gonic/gin`（`gin.TestMode`） |

### 命名规范

- **测试文件**：`{被测文件名}_test.go`，与被测源文件放在**同一目录、同一包**中
- **测试函数**：`Test{被测函数名}_{场景描述}`，如 `TestValidateRedirectURL`、`TestRefundTaskQuota_Wallet`
- **子测试**：通过 `t.Run("场景名", func(t *testing.T) {...})` 组织，场景名用英文或拼音短语描述
- **辅助函数**：以小写字母开头（包内私有），如 `seedUser`、`truncate`、`truncateTables`、`buildSSEBody`，并调用 `t.Helper()` 标记

## 运行测试

### 运行所有测试

```bash
go test ./...
```

### 运行特定包的测试

```bash

package common

import "testing"

func TestMyFunc_VariousInputs(t *testing.T) {
    tests := []struct {
        name    string
        input   string
        wantErr bool
        errMsg  string
    }{
        {name: "valid input", input: "ok", wantErr: false},
        {name: "empty input", input: "", wantErr: true, errMsg: "empty"},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            err := MyFunc(tt.input)
            if tt.wantErr {
                if err == nil {
                    t.Errorf("expected error, got nil")
                }
            } else if err != nil {
                t.Errorf("unexpected error: %v", err)
            }
        })
    }
}
```

        expected int
    }{

        })
    }
}

package service

import (
    "os"
    "testing"

    "github.com/QuantumNous/new-api/common"
    "github.com/QuantumNous/new-api/model"
    "github.com/glebarez/sqlite"
    "github.com/stretchr/testify/require"
    "gorm.io/gorm"
)

func TestMain(m *testing.M) {
    db, err := gorm.Open(sqlite.Open(":memory:"), &gorm.Config{})
    if err != nil {
        panic("failed to open test db: " + err.Error())
    }
    sqlDB, _ := db.DB()
    sqlDB.SetMaxOpenConns(1) // SQLite 内存库需限制为单连接

    model.DB = db
    common.UsingSQLite = true
    common.RedisEnabled = false

    if err := db.AutoMigrate(&model.Task{}, &model.User{}); err != nil {
        panic("migrate failed: " + err.Error())
    }
    os.Exit(m.Run())
}

func TestMyService_SomeOperation(t *testing.T) {
    // Arrange: 准备数据
    user := &model.User{Id: 1, Quota: 10000}
    require.NoError(t, model.DB.Create(user).Error)
    t.Cleanup(func() { model.DB.Exec("DELETE FROM users") })

    // Act: 执行被测逻辑
    MyService(user)

    // Assert: 验证结果
    var reloaded model.User
    require.NoError(t, model.DB.First(&reloaded, 1).Error)
    require.Equal(t, 10000, reloaded.Quota)
}

## 如何编写新测试

### 步骤 1：查找参考测试

```bash
# 查找针对特定函数/类型的现有测试
grep -r "TestRefund\|TestRecalc" new-api/service/

# 查找使用特定库的测试
grep -r "gin.CreateTestContext\|httptest.NewRecorder" new-api/relay/

# 列出某包下所有测试函数
grep -r "^func Test" new-api/relay/channel/
```

### 步骤 2：选择参考文件

**推荐参考文件**：

- `new-api/service/error_test.go` — 适用于**简单纯函数**，表格驱动 + `testify/require`
- `new-api/common/url_validator_test.go` — 适用于**工具函数**，多场景（成功/失败/边界）验证
- `new-api/relay/channel/gemini/relay_gemini_usage_test.go` — 适用于**需要 Gin 上下文的 HTTP Handler**
- `new-api/relay/helper/stream_scanner_test.go` — 适用于**并发/流式/性能**场景
- `new-api/service/task_billing_test.go` — 适用于**需要真实数据库的集成测试**（含 TestMain + seed helpers）
- `new-api/model/task_cas_test.go` — 适用于**数据库 CAS/并发写入**场景，及**model 包内集成测试**（需同时设置 DB 和 LOG_DB）
- `new-api/relay/channel/claude/message_delta_usage_patch_test.go` — 适用于**带子测试（t.Run）**的分组测试
- `new-api/relay/common/override_test.go` — 适用于**复杂 JSON 操作逻辑**，带条件和错误类型断言

### 步骤 3：编写测试

1. 在被测文件**同目录**下创建 `{filename}_test.go`，包名与被测包相同
2. 函数名格式：`Test{FuncName}_{ScenarioDesc}`
3. 对于纯函数：使用表格驱动（`[]struct{...}`）+ `for _, tc := range testCases { t.Run(...) }`
4. 对于需要 Gin 上下文的：在 `init()` 中调用 `gin.SetMode(gin.TestMode)` 或在每个测试开头调用
5. 对于依赖全局变量：使用 `t.Cleanup(func() { globalVar = orig })` 保存并还原
6. 对于通过 Getter 暴露的全局配置：直接修改 `GetXxx().FieldName`，同样用 `t.Cleanup` 还原
7. 对于需要数据库的：在包级定义 `TestMain`，用内存 SQLite 初始化 `model.DB`，并 `SetMaxOpenConns(1)`
8. 对于 `model` 包内的集成测试：同时设置 `model.DB` 和 `model.LOG_DB`
9. 对于接口依赖：在测试文件中定义 mock struct 实现接口，不引入外部 mock 框架
10. 并行测试：在函数顶部调用 `t.Parallel()`，子测试同理
11. 对于错误类型测试：使用包提供的 `AsXxx(err)` 辅助函数断言错误类型，不使用 `err.(*Type)` 直接类型断言

### 步骤 4：运行验证

```bash
# 运行新增测试文件所在包（加 -v 查看详情）
go test ./service/... -v -run TestMyNewFunc

# 确认无 race condition（集成/并发测试必做）
go test ./relay/helper/... -race -run TestStreamScanner

# 确认测试互不影响（运行整个包）
go test ./service/... -count=1
```

## 测试规范

### 命名规范

- 测试文件：`{source}_test.go`，与源文件同目录，同包名（如 `package service`，不用 `package service_test`）
- 测试函数：`Test{功能}_{场景}`，场景用下划线分隔，如 `TestRefundTaskQuota_Wallet`、`TestRefundTaskQuota_ZeroQuota`
- 辅助函数（Helper）：小写开头，首行调用 `t.Helper()`，如 `seedUser(t, id, quota)`、`truncateTables(t)`
- Mock 类型：`mock{接口名}`，如 `mockAdaptor`

### 测试结构

项目采用 **AAA 模式**（Arrange-Act-Assert）：

```
// Arrange: 准备测试数据和前置条件
seedUser(t, userID, initQuota)
seedToken(t, tokenID, userID, "sk-key", tokenRemain)
task := makeTask(...)

// Act: 执行被测逻辑
RefundTaskQuota(ctx, task, "reason")

// Assert: 验证结果
assert.Equal(t, initQuota+preConsumed, getUserQuota(t, userID))
log := getLastLog(t)
require.NotNil(t, log)
assert.Equal(t, model.LogTypeRefund, log.Type)
```

### 数据隔离规范

- 每个数据库集成测试调用 `truncate(t)` 或 `truncateTables(t)` 或通过 `t.Cleanup` 清理数据，防止测试间污染
- `t.Cleanup` 优先于 `defer`，因为它即使在子测试中也能正确执行
- 内存 SQLite 实例设置 `SetMaxOpenConns(1)` 避免 `SQLITE_BUSY` 错误

### Mock 原则

- **不引入 mock 框架**（如 gomock、mockery），而是在测试文件中手动实现接口
- Mock 只实现测试所需的方法，其余方法返回零值或 nil
- 全局变量/配置用 `orig := GlobalVar; t.Cleanup(func() { GlobalVar = orig })` 模式隔离，避免测试间相互干扰
- 外部 HTTP 依赖使用 `net/http/httptest.NewRecorder()` 和 `io.NopCloser` 构造假响应体

### Gin 上下文使用规范

- 包级 `init()` 中调用 `gin.SetMode(gin.TestMode)` 全局生效（参考 `new-api/relay/helper/stream_scanner_test.go:22`）
- 或在每个需要 Gin 的测试函数开头调用 `gin.SetMode(gin.TestMode)`
- 构造请求：`httptest.NewRequest(http.MethodPost, "/path", body)` + `gin.CreateTestContext(httptest.NewRecorder())`

```go
func TestUpdateWithStatus_ConcurrentWinner(t *testing.T) {
    const goroutines = 5
    wins := make([]bool, goroutines)
    var wg sync.WaitGroup
    wg.Add(goroutines)

    for i := 0; i < goroutines; i++ {
        go func(idx int) {
            defer wg.Done()
            won, err := task.UpdateWithStatus(TaskStatusInProgress)
            if err == nil { wins[idx] = won }
        }(i)
    }
    wg.Wait()

    winCount := 0
    for _, w := range wins { if w { winCount++ } }
    assert.Equal(t, 1, winCount, "exactly one goroutine should win the CAS")
}
```

并发测试要点：多 goroutine 同时操作，验证幂等性或唯一赢家语义。
