# 核心业务领域

> 本文档从用户视角描述服务提供的业务能力，说明用户可以通过本服务完成哪些业务任务。

## 服务定位

### 业务场景

new-api 是一个统一的 AI API 网关服务，面向需要调用多家 AI 提供商接口的个人开发者、企业和平台运营商。

**典型场景**:
1. **统一入口管理**: 运营商希望聚合 40+ 家 AI 提供商（OpenAI、Claude、Gemini、AWS Bedrock、Azure 等）的 API，对外暴露统一接口，用户无需关心底层具体由哪家提供商处理请求。
2. **多用户计费分发**: 平台运营商希望为多个用户或客户单独分配额度、令牌和访问权限，按使用量计费。
3. **成本控制与监控**: 企业希望追踪 AI API 的使用量、成本消耗，并对渠道进行健康检测和自动故障切换。
4. **接入多模态能力**: 用户需要调用文本对话、图像生成、音频转写、文本向量、视频生成、AI 绘画（Midjourney）、AI 音乐（Suno）等多种 AI 能力，通过单一平台统一访问。

### 解决的问题

- 用户不需要单独管理每家 AI 提供商的账号和 API Key，只需持有本平台颁发的令牌即可调用所有已接入的模型。
- 平台可以统一管控用户额度和权限，支持充值、订阅、兑换码等多种计费方式。
- 管理员可以配置多条渠道（同一提供商可配置多条），利用负载均衡、优先级和权重策略分散流量，提高可用性。

### 业务价值

- 降低接入多家 AI 提供商的开发和运维成本
- 实现用户级别的额度管控与使用审计
- 支持灵活的计费模式（按量付费、充值余额、订阅套餐、兑换码）
- 提供 OpenAI 兼容接口，现有使用 OpenAI SDK 的应用无需改造即可切换到本平台

## 核心业务概念

### 业务实体（用户视角）

| 实体名称 | 对用户的含义 | 用户关心的属性 | 示例 |
|---------|-------------|--------------|------|
| 用户（User） | 平台账号，代表一个使用者或客户 | 用户名、余额额度、所属分组、状态 | 普通用户、管理员账号 |
| 令牌（Token） | 用户创建的 API Key，用于调用 AI 接口 | 名称、额度上限、有效期、可用模型范围、IP 白名单 | sk-xxxxxxx |
| 渠道（Channel） | 管理员配置的上游 AI 提供商连接 | 提供商类型、支持模型、状态、余额、响应时间 | OpenAI GPT-4o 渠道、Claude 3.5 渠道 |
| 额度（Quota） | 平台内部计费单位，$1 ≈ 500,000 额度 | 剩余额度、已用额度 | 1,000,000 额度 ≈ $2 |
| 分组（Group） | 用户所属的权限分组，决定可访问的渠道和模型 | 分组名称 | default、vip、premium |
| 兑换码（Redemption） | 可兑换为额度的一次性码 | 码值、可兑换额度、有效期、状态 | 32 位随机字符串 |
| 充值记录（TopUp） | 用户通过支付渠道充值产生的订单 | 金额、支付方式、状态 | Stripe 支付、EPay 支付 |
| 订阅套餐（SubscriptionPlan） | 周期性付费套餐，提供定期配额 | 套餐名称、价格、有效期、每期额度 | 月度套餐、年度套餐 |
| 任务（Task） | 异步 AI 任务，如图像生成、音乐生成 | 任务平台、状态、进度、结果 URL | Midjourney 绘图任务、Suno 音乐任务 |
| 日志（Log） | 每次 API 调用产生的消费和操作记录 | 模型名称、消耗额度、Token 数、时间 | GPT-4o 对话消耗 100 额度 |
| 签到（Checkin） | 每日签到获取奖励额度 | 签到日期、获得额度 | 2026-03-02 签到获得 500 额度 |

### 业务术语（用户视角）

- **渠道（Channel）**: 管理员在后台配置的上游 AI 提供商连接，一个提供商可配置多条渠道，用于负载均衡或故障切换。
- **令牌（Token）**: 用户生成的 API 访问密钥（sk- 前缀），类似 OpenAI API Key，用于调用代理接口。
- **额度（Quota）**: 平台内部计量货币，$1 约等于 500,000 额度，用于统一衡量各模型的使用成本。
- **分组（Group）**: 控制用户可以访问哪些渠道和模型，管理员可为不同分组设置不同价格倍率。
- **模型定价（Pricing）**: 各模型的消费倍率，决定每次调用扣除的额度数量。
- **渠道亲和性（Channel Affinity）**: 某个令牌优先使用特定渠道的机制，提高响应一致性。
- **兑换码（Redemption Code）**: 预生成的一次性码，用户输入后可兑换为对应额度。
- **邀请码（AffCode）**: 用户专属推广码，被邀请注册的用户和邀请人均可获得额度奖励。

## 业务能力清单

### 核心业务能力

#### 1. AI 接口统一代理（API Relay）

**用户可以做什么**:
- 使用 OpenAI 兼容格式调用文本对话（Chat Completions）
- 使用 Claude 原生格式调用 Anthropic 模型
- 使用 Gemini 原生格式调用 Google 模型
- 调用图像生成（DALL-E、Stable Diffusion 等）
- 调用音频转写和语音合成（Whisper、TTS）
- 调用文本向量化（Embeddings）
- 调用文本重排序（Rerank）
- 调用实时语音对话（Realtime WebSocket）
- 调用 OpenAI Responses API

**使用场景**: 开发者将原来直接调用 OpenAI 的接口地址替换为本平台地址，其余代码无需改动，即可通过平台路由到多家提供商。

#### 2. 异步 AI 任务（Task/Midjourney/Suno）

**用户可以做什么**:
- 提交 Midjourney 绘图任务（imagine、change、blend、describe、edits、video 等操作）
- 提交 Suno 音乐生成任务（音乐创作、歌词生成）
- 查询任务状态和进度
- 获取任务结果（图片 URL、视频 URL、音频 URL）

**使用场景**: 需要生成图像或音乐的业务场景，任务异步执行，用户轮询或订阅结果。

#### 3. 令牌管理

**用户可以做什么**:
- 创建、编辑、删除 API 令牌
- 为令牌设置额度上限（或不限额度）
- 为令牌设置有效期（或永不过期）
- 限制令牌只能使用特定模型列表
- 限制令牌只能从特定 IP 地址调用
- 查询令牌使用情况和消费日志

**使用场景**: 为不同的业务系统、团队成员或客户创建独立令牌，实现访问隔离和独立计量。

#### 4. 额度充值与计费

**用户可以做什么**:
- 通过 Stripe（国际信用卡）充值额度
- 通过 EPay（国内支付）充值额度
- 通过 Creem 充值额度
- 输入兑换码兑换额度
- 查看充值历史记录
- 将邀请奖励额度转移到可用余额

**使用场景**: 用户余额不足时充值，或通过兑换码获取测试额度。

#### 5. 订阅套餐

**用户可以做什么**:
- 查看可购买的订阅套餐列表
- 购买订阅套餐（支持 Stripe、EPay、Creem）
- 查看当前订阅状态和剩余额度
- 设置订阅偏好

**使用场景**: 需要固定每月使用量的用户选择订阅套餐，获得周期性配额，不同套餐可升级用户分组权限。

#### 6. 日志与用量查询

**用户可以做什么**:
- 查看自己的 API 调用日志（模型、消耗额度、Token 数、时间）
- 按模型、令牌、时间筛选日志
- 查看每日用量趋势统计
- 通过令牌直接查询对应的调用记录

**使用场景**: 用户需要了解额度消耗情况，或排查调用异常时查阅日志。

#### 7. 每日签到

**用户可以做什么**:
- 每日完成签到，获取随机额度奖励
- 查看当月签到记录和历史签到统计

**使用场景**: 平台运营引导用户活跃，每日签到可获得额外免费额度。

## 用户角色和权限

### 角色说明

| 角色名称 | 角色描述 | 可以做什么 | 不能做什么 |
|---------|---------|-----------|-----------|
| 游客（Guest, role=0） | 未登录访问者 | 查看公开页面（公告、定价、关于） | 无法调用 API、无法查看个人数据 |
| 普通用户（User, role=1） | 已注册的标准用户 | 管理自己的令牌、查看自己的日志、充值额度、购买订阅、签到 | 无法管理他人账号、无法配置渠道或系统参数 |
| 管理员（Admin, role=10） | 平台运营管理员 | 管理所有用户、渠道、兑换码、日志；查看所有数据统计 | 无法修改系统全局配置（支付、OAuth、限流等） |
| 超级管理员（Root, role=100） | 系统所有者 | 所有操作，包括系统配置、性能管理、自定义 OAuth 提供商 | 无限制 |

### 权限边界

- **令牌访问控制**: 令牌调用 API 时受令牌自身的额度、有效期、IP 白名单、模型限制约束，与用户角色无关。
- **分组控制模型访问**: 用户所属分组决定可访问的渠道，同一渠道可限制只对特定分组开放。
- **令牌分组覆盖**: 令牌可指定独立分组，覆盖用户默认分组，实现更细粒度的访问控制。
- **跨组重试**: 在 auto 分组模式下，令牌可启用跨分组重试，自动在多个渠道组之间切换。

## 业务规则（用户需要知道的）

### 使用限制

- **令牌额度**: 令牌可设置剩余额度上限，耗尽后令牌状态变为 `Exhausted`，无法继续调用。
- **令牌有效期**: 令牌可设置过期时间，过期后状态变为 `Expired`，需重新创建或续期。
- **IP 白名单**: 令牌可绑定 IP 白名单，非白名单 IP 的请求将被拒绝。
- **模型限制**: 令牌可限定只能使用特定模型列表，调用其他模型将被拒绝。
- **全局限流**: 平台可配置 API 级别的全局限流策略，超出频率的请求返回 429 错误。
- **模型频率限制**: 同一令牌对同一模型的调用频率受平台模型级限流策略约束。
- **每日签到**: 每个用户每天只能签到一次，不可补签。
- **兑换码**: 每个兑换码只能使用一次，使用后状态变为已使用，不可重复兑换。

### 业务约束

- **用户名唯一**: 用户名在全平台唯一，不可重复注册。
- **邮箱唯一**: 同一邮箱地址只能绑定一个账号。
- **额度最小单位**: 邀请额度转入余额时，最低转移量为 1 个计费单元（$0.002），即 500,000 额度起转。
- **渠道自动禁用**: 若渠道连续失败次数超过阈值（默认 5 次），可配置自动禁用该渠道。
- **额度计算**: 调用完成后，实际消耗额度 = 模型基础倍率 × (输入 Token 数 × 输入价格 + 输出 Token 数 × 输出价格) × 分组倍率。
- **兑换码过期**: 兑换码可设置过期时间，过期后无法兑换。
- **订阅配额重置**: 订阅套餐可配置每日/每周/每月自动重置配额，到期重置（不累积）。
- **两步验证（2FA）**: 启用 2FA 后，登录时需要输入 TOTP 验证码或备用码。

### 状态流转（用户视角）

- **令牌状态**: `启用(Enabled)` → `禁用(Disabled)` / `过期(Expired)` / `额度耗尽(Exhausted)`
- **用户状态**: `启用(Enabled)` ↔ `禁用(Disabled)`（管理员可切换）
- **渠道状态**: `启用` ↔ `禁用`（手动或自动禁用）
- **兑换码状态**: `启用(Enabled)` → `已使用(Used)` / `禁用(Disabled)`
- **充值订单状态**: `待支付` → `已完成` / `失败`
- **订阅状态**: `active（生效中）` → `expired（已到期）` / `cancelled（已取消）`
- **任务状态**: `NOT_START` → `SUBMITTED` → `QUEUED` → `IN_PROGRESS` → `SUCCESS` / `FAILURE`

## 业务边界

### 本服务提供的能力

- 统一的 AI API 网关代理（OpenAI、Claude、Gemini 等兼容协议）
- 用户账号注册、登录、OAuth 第三方登录
- 多层次认证安全（2FA、Passkey、Turnstile）
- 用户额度管理与多种充值方式
- 订阅套餐管理与支付
- 令牌创建与精细化访问控制
- 渠道负载均衡、故障切换与健康检测
- 异步任务管理（Midjourney 绘图、Suno 音乐）
- 使用日志记录与查询
- 模型定价配置与查询
- 邀请推荐与奖励系统
- 每日签到奖励
- 兑换码管理

### 不提供的能力

- 不提供 AI 模型本身，需依赖外部上游提供商
- 不提供数据训练或微调能力（fine-tune 接口为占位符，返回未实现）
- 不提供文件存储服务（/v1/files 接口为占位符）
- 不提供模型的图像变体功能（images/variations 为占位符）

### 与其他服务的协同

- **上游 AI 提供商**: OpenAI、Anthropic、Google Gemini、AWS Bedrock、Azure OpenAI、阿里云百炼、百度千帆、智谱 AI、Minimax、DeepSeek、Kimi、SiliconFlow、VolcEngine、Coze、Dify、Replicate 等 40+ 家，需提前申请 API Key 并在渠道管理中配置。
- **支付服务**: Stripe（国际）、EPay（国内聚合支付）、Creem，需运营商自行接入对应支付商户账号。
- **OAuth 服务**: GitHub、Discord、OIDC、微信（需独立 Server）、Telegram、LinuxDO，需配置对应客户端 ID/Secret。
- **消息通知**: SMTP 邮件服务用于发送验证码和密码重置邮件，需配置 SMTP 服务器。
- **模型部署**: 支持通过 io.net 进行模型部署管理（IoNet 集成）。
