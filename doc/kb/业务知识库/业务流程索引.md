## 用户认证流程

### 1. 用户密码注册

**入口**: `POST /api/user/register` (`new-api/controller/user.go:Register`)

**核心步骤**:
1. 检查注册开关与邮箱验证码（如启用）
2. 检查用户名/邮箱是否已存在
3. 创建用户记录并关联邀请码
4. 可选：生成默认 API 令牌

**主要异常分支**:
- 注册功能关闭 → 返回 `注册功能已禁用`
- 用户名/邮箱已存在 → 返回 `用户已存在`
- 邮箱验证码错误 → 返回验证码错误

**涉及代码**:
- Handler: `new-api/controller/user.go:Register`
- DAO: `new-api/model/user.go`（`CheckUserExistOrDeleted`, `Insert`）
- DAO: `new-api/model/token.go`（`Insert`，生成默认令牌）

---

### 2. 用户密码登录

**入口**: `POST /api/user/login` (`new-api/controller/user.go:Login`)

**核心步骤**:
1. 验证账号密码（`model.User.ValidateAndFill`）
2. 检查是否开启 2FA；若是则保存 pending session 并要求二次验证
3. 未启用 2FA 则直接写入 session（id/username/role/status）并返回用户信息

**主要异常分支**:
- 密码登录功能关闭 → 返回禁用消息
- 密码错误 → 返回错误信息
- 2FA 验证失败 → 返回 2FA 验证失败

**涉及代码**:
- Handler: `new-api/controller/user.go:Login`, `setupLogin`
- DAO: `new-api/model/user.go:ValidateAndFill`
- Logic: `new-api/model/twofa.go:IsTwoFAEnabled`

---

### 3. 2FA 登录二次验证

**入口**: `POST /api/user/login/2fa` (`new-api/controller/twofa.go:Verify2FALogin`)

**核心步骤**:
1. 从 session 读取 pending_user_id
2. 验证 TOTP 码或备用码
3. 验证通过则完成 session 写入，登录成功

**主要异常分支**:
- session 中无 pending 用户 → 返回未授权
- TOTP 验证码错误 → 返回验证失败

**涉及代码**:
- Handler: `new-api/controller/twofa.go:Verify2FALogin`
- DAO: `new-api/model/twofa.go`

---

### 4. Passkey 登录（WebAuthn）

**入口**: `POST /api/user/passkey/login/begin` / `POST /api/user/passkey/login/finish` (`new-api/controller/passkey.go`)

**核心步骤**:
1. Begin：生成 WebAuthn Challenge 并存入 session
2. Finish：验证 WebAuthn 断言，匹配本地 Passkey 凭证
3. 验证通过后执行 `setupLogin` 完成会话建立

**主要异常分支**:
- Passkey 功能未启用 → 返回禁用错误
- 凭证不匹配 → 返回验证失败

**涉及代码**:
- Handler: `new-api/controller/passkey.go`
- Service: `new-api/service/passkey/`
- DAO: `new-api/model/passkey.go`

---

### 5. OAuth 标准登录/绑定（GitHub / Discord / OIDC / LinuxDO）

**入口**: `GET /api/oauth/:provider` (`new-api/controller/oauth.go:HandleOAuth`)

**核心步骤**:
1. 验证 CSRF state
2. 判断用户是否已登录：已登录则进入绑定流程，未登录进入注册/登录流程
3. 用 code 换取 provider access token，获取用户信息
4. 查找或创建本地用户，写入 session

**主要异常分支**:
- state 不匹配 → 返回 403 CSRF 错误
- provider 未启用 → 返回错误
- provider 返回错误码 → 返回 OAuth 错误

**涉及代码**:
- Handler: `new-api/controller/oauth.go:HandleOAuth`
- OAuth providers: `new-api/oauth/`

---

## 令牌（API Key）管理流程

### 16. 创建 API 令牌

**入口**: `POST /api/token/` (`new-api/controller/token.go:AddToken`)

**核心步骤**:
1. 验证令牌参数（名称长度、额度范围）
2. 检查用户令牌数量是否超上限
3. 生成随机 key，创建令牌记录

**主要异常分支**:
- 令牌数量超上限 → 返回限制错误
- 额度超出最大值 → 返回参数错误

**涉及代码**:
- Handler: `new-api/controller/token.go:AddToken`
- DAO: `new-api/model/token.go:Insert`

---

### 17. 更新 API 令牌

**入口**: `PUT /api/token/` (`new-api/controller/token.go:UpdateToken`)

**核心步骤**:
1. 验证令牌参数
2. 查询令牌（校验归属）
3. 若 status_only 则只更新状态，否则全量更新

**主要异常分支**:
- 令牌已过期/额度耗尽 → 无法重新启用
- 令牌不属于当前用户 → 404

**涉及代码**:
- Handler: `new-api/controller/token.go:UpdateToken`
- DAO: `new-api/model/token.go:Update`

**主要异常分支**:

**涉及代码**:

## AI 中继代理流程

### 33. OpenAI 聊天补全中继

**入口**: `POST /v1/chat/completions` (`new-api/controller/relay.go:Relay`)

**核心步骤**:
1. 解析并验证请求体（`helper.GetAndValidateRequest`）
2. 生成 RelayInfo（模型、分组、token 信息）
3. 敏感词检测（可选）、token 预估、预扣费
4. 选择满足条件的渠道，转发请求到上游
5. 收到响应后结算实际费用，记录日志

**主要异常分支**:
- 预扣费失败（额度不足）→ 返回 402
- 渠道不可用 → 自动重试其他渠道
- 上游返回错误 → 自动禁用渠道（如配置）并重试

**涉及代码**:
- Handler: `new-api/controller/relay.go:Relay`
- Helper: `new-api/relay/helper/`
- Service: `new-api/service/billing.go`, `new-api/service/quota.go`
- TextHelper: `new-api/relay/compatible_handler.go`
- DAO: `new-api/model/log.go`

---

### 34. Claude 格式中继

**入口**: `POST /v1/messages` (`new-api/controller/relay.go:Relay` with `RelayFormatClaude`)

**核心步骤**:
1. 以 Claude 格式解析请求
2. 通过 ClaudeHelper 转发到上游（可能适配到其他提供商）
