## 任务目标

生成外部接入指南，聚焦于：
- **服务暴露方式**：本仓库如何对外提供服务
- **外部调用规范**：外部调用应遵循的规范
- **认证与授权**：身份验证和权限控制
- **错误处理机制**：统一的错误码和处理建议
- **版本兼容性**：接口版本管理策略

**与 API 索引的区别**：
- **API 索引**：关注**接口定义**（所有 API 的列表、参数、位置）
- **外部接入指南**：关注**如何调用**（集成方式、SDK 使用、认证流程）

## 注意事项

1. **清晰性与准确性**：
   - 描述应清晰、简洁，避免歧义。
   - 提供的代码示例应可运行、正确。

2. **全面性**：
   - 覆盖所有对外暴露的接口类型 (HTTP, gRPC, MQ, 公共库/SDK)
   - 对于**公共库类型项目**，必须详细列出所有导出的函数、类、接口、常量
   - 提供完整的导出项列表，包含签名、功能说明、使用示例
   - 包含认证、授权、错误处理、版本管理等关键方面

3. **易用性**：
   - 站在外部调用方的角度，提供他们最关心的信息。
   - 提供不同编程语言的调用示例 (**HTTP API, gRPC, SDK**)，降低集成难度。

4. **可维护性**：
   - 指南应易于更新，反映接口的最新状态。

5. **路径规范**：引用代码或配置时使用相对路径格式 `项目目录/文件路径`。**必须**使用任务上下文中提供的 `项目目录` 值作为路径前缀，不要使用绝对路径，也不要省略项目目录前缀

6. **执行原则**: 你的上下文窗口会在接近限制时自动被压缩，因此不要因为Token预算问题提前停止任务，即使预算快用完，也要尽可能完整执行任务。

7. **公共库扫描完整性**（如适用）：
   - 必须扫描所有导出文件，确保不遗漏任何公共函数、类、接口
   - 优先扫描主导出文件（如 index、lib、__init__ 等）
   - 按功能模块分组展示导出项，便于查找
   - 提供实用的代码示例，降低使用门槛
